About Webui

Webui is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. Webui has been build from the ground up with flexibility as the primary goal - even the components are made that way. Webui is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Toggle menu.
 

Toggle Container[Requires: webui.js]

The toggle container is a generic component used to manage the showing and hiding of elements, but it is a bit more than that. It combines funtionality found in menus and tabs, but adds various behaviours such as inclusive and exclusive behaviour, off-canvas shifting, and transitions. The result is a component that allows you to invent new ways to present web content. The hope is that new and better capabilities will be added to this component over time, and that it becomes much more than the sum of it's parts.

Right now, it is a trivial task to setup off-canvas menus, fly-out menus, and accordions, etc. It is important to know the patterns to use for each type of component you want. Here are some patterns to implement various common components.

Off-canvas menus

A basic template to create a left side off-canvas menu is as follows:

<div class="toggle-container" data-fade-in-duration="600" data-fade-out-duration="600" id="toggle1">

  <div class="content-col dock-top-left">
    <button data-target="#panel1" class="toggle-activator menu-button-sm width-sm"><i class="fa fa-chevron-right"></i></button>
  </div>

  <div class="content-col dock-top-left">

    <div class="toggle-item light border-sm border-secondary off-canvas-left" id="panel1">
                        
      <div class="content-col place-top-left">
        <button data-target="#panel1" class="toggle-activator menu-button-sm width-sm"><i class="fa fa-chevron-left"></i></button>
      </div>        
            
      <div class="area width-lg">
                
        <!-- Menu content goes here -->
                
      </div>            
            
    </div>
  </div>
</div>

You don't need to worry about button styles and colors, that's up to you. But the structure must be as above. You can also create a right side off-canvas by simply changing the left classes to right - the toggle container will take care of the rest.

NOTE:

For off-canvas menus you do need to add class="off-canvas-body" to the body tag in your html.

Fly-out menus

A basic template to create a right side fly-out menu is as follows:

<div class="toggle-container" data-fade-in-duration="600" data-fade-out-duration="600" id="toggle1">
        
  <div class="content-col dock-top-right">
    <button data-target="#panel1" class="toggle-activator menu-button-sm width-sm"><i class="fa fa-chevron-left"></i></button>
  </div>

  <div class="content-col dock-top-right">

    <div class="toggle-item dark hidden place-top-right" id="panel1">
        
      <div class="content-col place-top-right">
        <button data-target="#panel1" class="toggle-activator menu-button-sm width-sm"><i class="fa fa-chevron-right"></i></button>
      </div>        
        
      <div class="area text-xl">
        <!-- menu content goes here -->
      </div>
                            
    </div>
  </div>
</div>

One important difference with the fly-out menu is that the toggle-item is explicitly hidden using the hidden class - this is because the toggle container handles fly-out menus in a more generic way, so is not concerned with the initial visibility. It is possible that this could change in the future.

Accordions

A basic template to create an accordion menu is as follows:

<div class="toggle-container" data-fade-in-duration="600" data-fade-out-duration="600" id="toggle1">
            
  <div class="content-col-20">
    
    <button data-target="#panel1" class="toggle-activator menu-button menu-button-accent-2">Heading 1</button>
    <div id="panel1" class="toggle-item width-full hidden">
      <div class="panel primary border-sm border-accent-2 height-lg width-full">
        <!-- Content for panel 1 -->
      </div>
    </div>
        
    <div class="row-spacing-xs"></div>
        
    <button data-target="#panel2" class="toggle-activator menu-button menu-button-accent-2">Heading 2</button>
    <div id="panel2" class="toggle-item width-full hidden">
      <div class="panel primary border-sm border-accent-2 height-lg width-full">
        <!-- Content for panel 2 -->
      </div>
    </div>
        
    <div class="row-spacing-xs"></div>
        
    <button data-target="#panel3" class="toggle-activator menu-button menu-button-accent-2">Heading 3</button>
    <div id="panel3" class="toggle-item width-full hidden">
      <div class="panel primary border-sm border-accent-2 height-lg width-full">
        <!-- Content for panel 3 -->
      </div>
    </div>

  </div>        
</div>

Events

There are 4 event callbacks that can be used, ui.toggle.show.before, ui.toggle.show.after, ui.toggle.hide.before, and ui.toggle.hide.after. The following script shows how to setup the event callbacks.

$( document ).ready(function() {

  $( "#panel1" ).on("ui.toggleItem.show.before", function(e) {
    // Take some action
  });

  $( "#panel1" ).on("ui.toggleItem.show.after", function(e) {
    // Take some action
  });

  $( "#panel1" ).on("ui.toggleItem.hide.before", function(e) {
    // Take some action
  });

  $( "#panel1" ).on("ui.toggleItem.hide.after", function(e) {
    // Take some action
  });

});
