About WebUI

WebUI is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. WebUI has been build from the ground up with flexibility as the primary goal - even the components are made that way. WebUI is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Shapes [Requires: webui.js]

WebUI supports both transform shapes and shapes making use of clip-path. Transform shapes that work by transforming multiple div elements are widely supported by modern browsers. But although clip-path shapes are more powerful, they are not supported by Internet Explorer or Microsoft Edge. Both types of shapes are provided with a set of default ready-made shapes and also a single custom shape facility.

NOTE:

The new snap positioning tool is an ideal way to position shapes relative to any other element used as an anchor point. See the DEMO-Grids-Positioning.html demo page with the install or download package.

Transform shapes

The screenshot above shows the pre-defined transform shapes available and also a custom shape example. Creating a transform shape requires just a single div element and a few data dash attributes. Below is an example of a flat diamond shape. 

<div class="diamond-flat-shape align-center" data-size="5rem" data-rotation="0" data-class="custom-image-2"></div>

The pre-defined shape classes are:

  • isosceles-shape
  • rhombus-shape
  • parallelogram-shape
  • diamond-shape
  • hexagon-shape
  • octagon-shape
  • diamond-flat-shape
  • scalene-left-shape
  • scalene-right-shape

The data dash parameters are as follows:

data-size As with css styles, the size and unit type must be specified together - you can use any type of unit supported by CSS 3.
data-rotation This parameter is used to specify the degree of rotation required for the whole shape. When using rotation you may wish to use an appropriately rotated image, so that it remains upright within the shape.
data-class Use data-class to specify any CSS class you may have, such as an image or color. This will apply the class to the background of the shape. When using a background image it is important to use a suitable size and ensure that it is a true square - this will prevent the image from being stretched or shrunk. The image can be any size within reason, but you will want to ensure it does not get unnecessarily stretched or distorted. The easiest way is to make the image match the size specified for the shape itself. Transform shapes only allow a background image to be set, rather than an img element nested inside the shape.

Custom transform shapes using transform modifier data

By specifying the class custom-shape instead of one of the pre-defined shapes, and using an additional parameter data-modifiers, you can create your own custom shapes by experimenting with individual values. However, if you are familiar enough with how CSS transforms behave, then you will have a head start.

Here is an example that creates a trapezium shape. 

<div class="custom-shape" data-size="5rem" data-rotation="0" 
  data-modifiers="0.9,0.8,-110,-.05,0,0.8,0.9,110,.1,.12,1,1,0,-0.1,-0.1" data-class="custom-image-2"></div>

Explanation of each transform modifier value

Modifier values represent 3 groups of 5 values. The first group controls the shape container div or the outer most div (Not the div you specify in your markup, but it's immediate child'). The second group controls the inner div or the child of the container. Finally, the third group controls the content div or the inner most div.

Modifier value 1: The multiplying factor of the container width relative to the shape size value.
Modifier value 2: The multiplying factor of the container height relative to the shape size value.
Modifier value 3: The degree of rotation (CSS rotate property) of the container div.
Modifier value 4: The multiplying factor of the container X-axis movement (CSS translateX property) relative to the shape size value.
Modifier value 5: The multiplying factor of the container Y-axis movement (CSS translateY property) relative to the shape size value.

Modifier value 6: The multiplying factor of the inner div width relative to the shape size value.
Modifier value 7: The multiplying factor of the inner div height relative to the shape size value.
Modifier value 8: The degree of rotation (CSS rotate property) of the inner div.
Modifier value 9: The multiplying factor of the inner div X-axis movement (CSS translateX property) relative to the shape size value.
Modifier value 10: The multiplying factor of the inner div Y-axis movement (CSS translateY property) relative to the shape size value.

Modifier value 11: The multiplying factor of the content div width relative to the shape size value.
Modifier value 12: The multiplying factor of the content div height relative to the shape size value.
Modifier value 13: The degree of rotation (CSS rotate property) of the content div.
Modifier value 14: The multiplying factor of the content div X-axis movement (CSS translateX property) relative to the shape size value.
Modifier value 15: The multiplying factor of the content div Y-axis movement (CSS translateY property) relative to the shape size value.

CSS clip-path shapes (Not supported in Internet Explorer or Microsoft Edge)

The screenshot above shows the pre-defined clip-path shapes available and also a custom shape example. Creating a clip-path shape requires just a single div element, a specific shape class, and any WebUI classes that you might normally use to style elements, such as width, height, alignment, and background. Below is an example of a clip-path kite shape. Clip-path shapes allow you to use an image element nested inside the shape div if you prefer that approach, or you can apply the clip-path shape classes directly to the img element. 

<div class="width-md height-md align-center custom-image-2 kite-clip-shape"></div>

The pre-defined clip-path shape classes are:

  • rhombus-clip-shape
  • rhomboid-clip-shape
  • kite-clip-shape
  • trapezoid-isosceles-clip-shape
  • triangle-isosceles-clip-shape
  • pentagon-clip-shape
  • star-clip-shape

Custom clip-path shapes using polygon points data

Custom clip-path shapes are specified differently to transform shapes and require an additional javascript call.

Here is an example that creates a custom scalene triangle with polygon point pairs. 

<div class="width-md height-md align-center custom-image-2" id="customClipShape"></div>

<script>
  webui.ready(function() {

    ui("#customClipShape").renderPolygonShape("0.2 0.7, 0.5 0, 0.5 0, 1 1");
	
  });
</script>

Razor-blade style

Razor-blade styles are a great way to show off sections on your website by giving an angular shape to elements such as a div. The classes available for this are:

  • razor-blade-top-left
  • razor-blade-top-right
  • razor-blade-bottom-left
  • razor-blade-bottom-right

You also need to add one of the following classes to specify the amount of angle needed:

  • razor-blade-sm
  • razor-blade-md
  • razor-blade-lg

Arrow extend style

The arrow extend classes allow you to easily extend most elements with an arrow extension. The border of the element is also extended into an arrow shape. The classes available for this are:

  • arrow-extend-left
  • arrow-extend-top
  • arrow-extend-right
  • arrow-extend-bottom

You also need to add one of the following classes to specify the size of the arrow:

  • arrow-extend-sm
  • arrow-extend-md
  • arrow-extend-lg
