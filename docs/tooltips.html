Using tooltips

Tooltips are a popular way of displaying hints to the user, and can help guide and make the user's experience more intuitive. Webui supports customisable tooltips and the following screenshot shows the available sizes, small, medium and large. Although an initial size is set on the tooltip they do automatically downsize for smaller screen areas, so a tooltip that is initially large may switch to medium or small when displayed in a confined space. Automatic resizing and re-positioning can be enabled or disabled along with other configuration options as explained a bit later. You will need the webui.js javascript file included in your project for tooltips to work.

<div class="content-row">
  <div class="content-col-20 col-spacing-left-xl">
    <span class="tooltip" id="tooltip1">
      <span class="tooltip-dynamic tooltip-animate tooltip-accent-1 tooltip-bottom tooltip-sm rounded-md">Example tooltip.</span>
      <input id="FirstName" type="text" class="input-sm rounded-sm" />
    </span>         
  </div>
</div>

On line 03 above, the class tooltip defines a tooltip wrapper for the tooltip itself and the tooltip target control. The target in this case is an html input. On line 04 (the actual tooltip), the tooltip-dynamic class makes the tooltip show on hovering over the input field, and then tooltip-accent-1 is setting the theme colour for the tooltip. Also, the classes tooltip-bottom and tooltip-sm define the positioning and size.

You can choose the tooltip behaviour by using the following classes:

tooltip-dynamic (Causes the tooltip to show while hovering over it's target control)

tooltip-static (Causes the tooltip to show on focus of it's target control)

tooltip-animate (Causes the tooltip to fade in when activated and fade out when de-activated)

The positioning of tooltips is controlled by the following classes:

  • tooltip-top
  • tooltip-left
  • tooltip-bottom
  • tooltip-right

The width of tooltips is controlled by the following classes:

  • tooltip-sm
  • tooltip-md
  • tooltip-lg

There may be cases when you need to show or hide tooltips via javascript. You can do that simply by specifying an id on the tooltip wrapper, as shown above, and calling one of the webui.js functions, and optionally specifying a message. Below is some javascript to configure options and explicity show a tooltip.

$( document ).ready(function() {

  ui.initTooltips({
    autoPositioning: true,
    autoResizing: true,
    autoPositioningMargin: 0
  });
  
  ui.showTooltip($("#tooltip1"));
	
});

Tooltip resizing and positioning

Tooltips use viewport boundry collision detection to attempt to avoid the possibility of being placed off screen or partially off screen. On larger screens and when the browser window is resized or scrolled the re-positioning functionality is dominant, but as the screen area gets smaller the tooltip re-sizing becomes more dominant with re-positioning more restrictive. What that means is that on a phone for example, the tooltip would likely stay at the smallest width and normally be located at the top or bottom of the target element. If, on the other hand the web page was viewed on a desktop screen, tooltips would be much more likely to stay at their original size.

Customisation

As with all components in Webui, tooltips may be customised in a number of ways. However, they are a bit limited when larger amounts of content are required due to pre-defined widths and padding. This is not going to be a problem as Webui is flexible enough to allow hybrid components, where we mix the functionality from various components.

Considering the limitations of tooltips, let's look at some way of displaying content that partially behaves like a tooltip and a drop down menu combined. Take a look at the following screenshot.

Because the class menu-activator-dynamic has been applied to the textbox on line 01, it means that the dropdown will show on hovering over the textbox. Also, as we have the menu-close class on the dropdown content and each image is wrapped in an anchor tag, the dropdown will hide when an image is clicked. Of course, we get all of the other behaviour associated with dropdowns such as fade-in and fade-out effects, and inclusive and exclusive behaviours as part of the package.
