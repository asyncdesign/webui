About WebUI

WebUI is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. WebUI has been build from the ground up with flexibility as the primary goal - even the components are made that way. WebUI is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Flexbox

Overview

Flexbox was designed to improve the arrangement and layout of content whatever the size or quantity, but was never intended to be a replacement for grid systems, and can be introduced to existing markup as an enhancement. Flexbox is by no means an all or nothing solution, and was specifically given the capability to override existing CSS behaviours in many cases.

So how does it work? Flexbox is designed around the idea of a flex container (the parent element) and flex items (the child elements). The browser will calculate when a flex item should be repositioned, resized, or wrapped (if flex-wrap is enabled). This is based on the flex container size, available space within it, and the size of flex items.

It is possible to layout an entire site using primarily flexbox, due to some of it's unique capabilities, but you should continue to use traditional techniques where it makes sense. Flexbox is all about content driven design, where contained html elements can stretch, grow, shrink, or wrap, depending on available space. You might then ask, so how does this play in terms of a responsive web design? Given a flex container and a set of flex items within the container, you can construct a fully responsive layout without the need for media queries or breakpoints - however, media queries may be needed for more complex layouts or device related requirements, for example. Flexbox excels when it comes to alignment and spacing, which can dramatically reduce the dependence on margin and padding adjustments. It is a great solution for arranging html elements within components.

WebUI supports the full range of flexbox features, providing a comprehensive set of utility classes, including media breakpoint specific equivalents.

Naming convention

All flexbox related classes are prefixed by flex-. This avoids any confusion between flexbox and non-flexbox classes.

Basic structure

The code snippet below shows how to make a very simple flexbox structure using some of the basic flexbox utility classes. The flex container is nothing more than a div being set to display: flex and using the defaults for flex-grow, flex-shrink, and flex-basis - in most cases this is sufficient for the container. Within that are two div tags, which automatically become flex items because they are immediate children of the flex container. The flex items have also been set to flex-grow so that they will each expand equally to use all available space in the flex container.

<div class="flex">
  <div class="flex-grow border-sm border-accent-2 background-secondary pad-sm">
    Flex item 1
  </div>
  <div class="flex-grow border-sm border-accent-2 background-secondary pad-sm">
    Flex item 2
  </div>
</div>

Another way to do the same thing is by using higher level classes such as flex-row and flex-column.

<div class="flex-row">
  <div class="flex-col border-sm border-accent-2 background-secondary pad-sm">
    Flex item 1
  </div>
  <div class="flex-col border-sm border-accent-2 background-secondary pad-sm">
    Flex item 2
  </div>
</div>

Flex item sizing

Much like the grid system, flex items can be given a size ranging from 1 to 20. Although they perform a similar function, the difference is that flex item sizes can be either vertical or horizontal, depending on the flex direction of the flex container. You can change the flex direction by using either flex-direction-row or flex-direction-column on the container.

The code below shows how to use flex item sizes.

<div class="flex-row border-sm border-muted pad-sm">
  <div class="flex-size-6">     
    <div class="flex-col pad-xs border-sm border-accent-2 background-secondary">
      Lorem ipsum dolor sit amet        
    </div>
  </div>
  <div class="flex-size-14">
    <div class="flex-col pad-xs border-sm border-accent-2 background-secondary">
      Lorem ipsum dolor sit amet
    </div>
  </div>
</div>

Media breakpoint utilities

Almost all WebUI utility classes will have corresponding breakpoint specific versions.

Naming convention

Breakpoints are numbered 1 to 5, with 1 being the smallest. A breakpoint suffix can be added to any utility class using the format -bp-[breakpoint number]-[direction]. The direction is either under or over

. For example -bp-1-under or -bp-3-over. 
<div class="flex-row border-sm border-muted pad-sm">
  <div class="flex-size-6 flex-size-14-bp-4-under flex-size-10-bp-2-under">     
    <div class="flex-col pad-xs border-sm border-accent-2 background-secondary">
      Lorem ipsum dolor sit amet        
    </div>
  </div>
  <div class="flex-size-14 flex-size-6-bp-4-under flex-size-10-bp-2-under">
    <div class="flex-col pad-xs border-sm border-accent-2 background-secondary">
      Lorem ipsum dolor sit amet
    </div>
  </div>
</div>

In the example above, each flex item has one default size applied and two further breakpoint sizes - this will cause each flex item to change size at each of those breakpoints. They will start off at 6 and 14 making a total of 20 for the largest screen size, then swap sizes at breakpoint 4, and change to 2 equal sizes at breakpoint 2.

More complex objects

This snippet shows the use of WebUI flexbox to layout a typical responsive navbar

<div class="panel background-dark">
  <div class="flex-row flex-items-center flex-justify-space-between flex-direction-column-bp-1-under">
    <div class="flex-initial flex-justify-center-bp-1-under">
      <a href="javascript:void 0" class="color-accent-2 text-xl">COMPANY LOGO</a>
    </div>              
    <div class="flex-auto flex-wrap flex-justify-center flex-justify-space-around flex-direction-column-bp-1-under">
      <a href="#0" class="text-sm color-light color-hover-success col-spacing-sm">PAGES</a>
      <a href="#0" class="text-sm color-light color-hover-success col-spacing-sm">PROMOTIONS</a>
      <a href="#0" class="text-sm color-light color-hover-success col-spacing-sm">PRODUCTS</a>
    </div>          
    <div class="flex-initial flex-justify-end-bp-1-over">
      <input type="text" class="input-md width-full" placeholder="Search" />
    </div>
  </div>
</div>

On the second line in code example above, a flex-row is defined (this is the flex container), then the flex-items-center makes all flex items (the flex children), center vertically - this only applies to the immediate children of the container and in this case it's the 3 div tags. Next, flex-justify-space-between adds equal spacing between the flex items. The last class applied to the container is flex-direction-column-bp-1-under, and you will notice that it has the suffix of -bp-1-under - this suffix only allows the class to apply at the smallest media breakpoint and under, so what we are doing here is effectively turning the row into a column by changing the flex direction at that breakpoint.

There are 3 flex items defined, the first and last have the class flex-initial applied, and the second flex item has the class flex-auto. flex-initial is the default for flexbox flex items, but not for WebUI flex items - what this means is that it cannot grow, but is allowed to shrink if required and will be just large enough to accomodate its content. flex-auto on the other hand, is similar except that it is allowed to grow if required to fill any remaining space - this is the default for WebUI flex-items.

Because all of the flex items within the container also have flexbox classes added, it means that their immediate children have now become flex items also - in this case its the actual links and text elements.
