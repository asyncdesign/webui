About Webui

Webui is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. Webui has been build from the ground up with flexibility as the primary goal - even the components are made that way. Webui is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Tabs[Requires: webui.js]

Webui attempts to make tabs easy to customise, not limited to links or ordered and unordered lists. With this tab control you can use any link or button type, or even any html element for the navigation. Below is an example of using links styled as menu buttons.

Here's the markup to create the above tabs layout.

<div id="tabs1" class="tabs width-xl">

  <div class="content-row">         
    <div class="content-col col-spacing-right-xs">
      <a href="#tab1" class="tab-activator menu-button-sm menu-button-link-sm menu-button-accent-1 rounded-top-md width-md">Tab 1</a>
    </div>
    <div class="content-col col-spacing-right-xs">
      <a href="#tab2" class="tab-activator menu-button-sm menu-button-link-sm menu-button-accent-1 rounded-top-md width-md">Tab 2</a>
    </div>
    <div class="content-col col-spacing-right-xs">
      <a href="#tab3" class="tab-activator menu-button-sm menu-button-link-sm menu-button-accent-1 rounded-top-md width-md">Tab 3</a>
    </div>          
  </div>
                
  <div class="content-row border-sm border-accent-2">
    <div class="tab-item panel primary height-7" id="tab1">
      Tab 1 content
    </div>
    <div class="tab-item panel primary height-7 hidden" id="tab2">
      Tab 2 content
    </div>
    <div class="tab-item panel primary height-7 hidden" id="tab3">
      Tab 3 content
    </div>
  </div>
                
</div>

No addition code is needed for fully working tabs and the above code is all that is required. However, if you want to select a default tab you can just do the following. 

$( document ).ready(function() {

  $("#tabs1").tabControl({ 
    activeTabId: "#tab2", 
    activeTabFocused: true 
  });
	
});

There are many possible customisations that can be done such as side or bottom tabs, dropdowns, badges, etc.

Events

There are 2 event callbacks that can be used, ui.tabs.change.before and ui.tabs.change.after. The before event is fired just before the tab change occurs, and the after event is fired just after the tab has changed. The following script shows how to setup the event callbacks.

$( document ).ready(function() {

  $( "#tabs1" ).on("ui.tabs.change.before", function(e, prevTabId, tabId) {
    // Take some action
  });

  $( "#tabs1" ).on("ui.tabs.change.after", function(e, prevTabId, tabId) {
    // Take some action
  });
	
});

When selecting a new tab the before and after events are raised, and you can get the id of the previous and newly selected tabs, since they will be passed into the callback for you. If no previous tab was selected when a tab change occurs, then the prevTabId will be undefined. You might want to handle that scenario.
