About Webui

Webui offers an alternative to larger frameworks, providing a small, flexible, efficient and easy to use system to create websites quickly. The philosophy is to follow a "Simplicity is the ultimate sophistication" approach, providing simple building blocks to construct much larger CSS components without the pain. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Webui allows you to create your web app without getting distracted by complexity.

Current version: 4.3.1.

Setting up a modal dialog

Webui allows you to easily create responsive model dialogs in 3 different sizes: small, medium, and large. Modal dialogs are positioned at the center of the screen and will be fixed in that position regardless of the browser viewport. You can override the positioning using your own css or by using Webui positioning classes. These dialogs are responsive out of the box, so the width and height will adjust according to the available screen area - this is so that a dialog can never go off screen. As far as the dialog content goes there is no special layout for dialogs, so you can design your dialog how you want it. However, it's always best to design your layout to suit the initial dialog sizes. But if you need more height and you don't mind a vertical scrollbar, then you can add as much as you wish.

Let's go through the code above. The first line is where the wrapper is defined for the modal - that's actually the overlay that you see covering the page behind. Then on line 02, the actual modal dialog is defined at the size we want, in this case the class dialog-md is a medium size. You may notice the dialog-animate class also, which just makes the dialog fade in and out. The final part is the dialog content, where the class dialog-content is applied. There is also a backgound color set using the primary theme color, in this case it's white. The rest of the markup is entirely your choice, but it's worth taking a closer look at how we open and close the dialog and at some of the classes used in the above example.

So to open the dialog it's just a case of adding an id to the outer wrapper and calling ui.toggleControl and passing in the id or the object, or you can just use jquery to show it. You will also see that ui.toggleControl is used for closing the dialog too.

In the above example, there are a few spacing and positioning classes of interest. On line 06 the class col-spacing-left-sm is used - this just moves the text away from the left of the heading, and we do something similar for the small X button in the top right. Finally, an important class used here is place-bottom-right - this positions the buttons at the bottom right of the dialog, but some space is needed between the buttons and the bottom edge, so just add some row spacing. If the content of the dialog is longer than the dialog itself, the positioned buttons will stay exactly where you put them relative to the dialog view window.
