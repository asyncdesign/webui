Using theme colours

Webui makes life a little easier when it comes to applying colour to your website. Firstly, there are a number of theme colours already defined as variables in the included SASS file. In total, there are 15 main CSS colour classes as follows:
  • default
  • primary
  • secondary
  • accent_1
  • accent_2
  • accent_3
  • accent_4
  • success
  • info
  • warning
  • danger
  • highlight
  • transparent
  • muted
  • disabled
These main theme colours can be applied to any element. Each of the above main colours also has a corresponding foreground colour and a set of behavioural colours as follows:
  • hover
  • active
  • border
  • foreground
However, these sub colours are not directly useable, but are used by other utility classes such as button-success or menu-button-accent-1 for their hover and active colours for example. This is done in order to maintain theme consistency. Here's an example of how colours may be used:

<div class="panel background-accent-1 rounded-md">
  <div class="form-row">
    <div class="form-col-20 align-center">
      <a href="#">        
        <div class="area rounded-md primary background-hover-secondary">
          <i class="fa fa-user fa-5x"></i>
        </div>          
      </a>
    </div>
  </div>
</div>
Here you will see that the panel container is being set with a background colour of background-accent-1, which is defined in the Webui CSS file. It is also being given a medium rounded appearence. On line 05 you will notice that the nested area container is being set with a primary background colour and primary foreground colour using the single primary class. Then the class background-hover-secondary makes the background change to the secondary theme colour when hovering over it.
