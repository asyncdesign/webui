About Webui

Webui is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. Webui has been build from the ground up with flexibility as the primary goal - even the components are made that way. Webui is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Theme Colours

Webui makes life a little easier when it comes to applying colour to your website. Firstly, there are a number of theme colours already defined as variables in the included SASS file. In total, there are 17 main CSS colour classes as follows:

Default
Primary
Secondary
Dark
Accent-1
Accent 2
Accent 3
Accent 4
Accent 5
Success
Info
Warning
Danger
Highlight
Transparent
Muted
Disabled

These main theme colours above can be applied to any element. Each of these colours also has a corresponding foreground colour, border colour, and a set of behavioural colours as follows:

  • hover
  • active
  • border
  • foreground

The behavioural classes are used by other utility classes such as button-success or menu-button-accent-1 for their hover and active colours, but you can override the defaults by using them as you would any other colour classes.

Here's an example of how colours may be used: 
<div class="panel background-accent-1 rounded-md">
  <div class="form-row">
    <div class="form-col-20 text-center">
      <a href="#">        
        <div class="area rounded-md primary background-hover-secondary">
          <i class="fa fa-user fa-5x"></i>
        </div>          
      </a>
    </div>
  </div>
</div>
Here you will see that the panel container is being set with a background colour of background-accent-1, which is defined in the Webui CSS file. It is also being given a medium rounded appearance. On line 5 you will notice that the nested area container is being set with a primary background colour and primary foreground colour using the single primary class. Then the class background-hover-secondary makes the background change to the secondary theme colour when hovering over it.
