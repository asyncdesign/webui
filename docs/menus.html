Creating menus

Webui places a lot of emphasis on the ability to create menus in the style that you want. As you will see, there are a lot of customisation options available. Menus do require the webui.js javascript file to be included in your project as well as the CSS, but you won't have to write any javascript except for setting configuration options as explained a bit later. First, let's take a look at some simple menus and then progress to the more advanced features.

The code example above shows a basic menu for a typical desktop layout, where a number of fixed widths are used and we haven't added any media flags for responsive behaviour. Webui allows the use of almost any html elements to build a menu, and you are not limited to buttons or lists. It is worth noting that Webui includes menu button types that are specifically styled and sized to suit menu layouts Let's examine the important parts as follows:

menu-bar-sm The are two menu-bar classes - menu-bar and menu-bar-sm. You don't always need them, but it's as well to add one of them, since they set a height and maximum height on the menu bar to match the height of corresponding controls that may be used. Of course, if you have your own custom elements that don't fit those dimensions, then you probably don't want it.

menu-activator This class registers a control as the element that will be the trigger to open and close a menu. You can use just about any element as the menu activator.

dropdown-content The dropdown-content class registers an element to be the container for any dropdown content. The dropdown content itself can be anything you want.

menu-close Specifying menu-close on the dropdown container will make the dropdown close after a menu item such as a button or link is clicked or tapped. If you happened to have content other than buttons and links and you want to toggle it via script, you can do that by calling the toggleMenuItem Webui function and passing in the menu activator element id or as an object.

On line 05 a the class dropdown-content-expand is added - this is actually the class that will make the menu push any page content down as you expand the menu. If you prefer the menu to overlay any page content you can replace just that single top most dropdown-content-expand with dropdown-content. There is a third kind of dropdown container called dropdown-sheet - this is used mainly for desktop sized screens and is often used on eCommerce sites.

NOTE: If you do use dropdown-content instead of dropdown-content-expand on the top most activator button for a mobile menu, you will need to add the menu-inclusive class as well, otherwise the menu will close when selecting sub menu activators. That's because dropdown-content on it's own will close all other menus by default and would normally be used for the desktop style menu.

Customisation

OK, now let's do some customisation of our first menu. Say we want to have a white dropdown with a small arrow on top, and maybe a black border and make the corners a bit rounded. Oh, and let's make it dropdown on hover instead if the default click. No problem, taking the first menu section (lines 06 - 15) from our first example we can change it as follows:

<div class="content-col col-spacing-left-sm width-5">
  <button id="Menu1" type="button" class="menu-activator-dynamic menu-button-sm menu-button-default align-left">Option 1<i class="fa fa-caret-down col-spacing-left-xs"></i></button>
  <div class="dropdown-content menu-close width-10 transparent">
    <div class="col-spacing-left-md">
      <div class="pointer-top-sm pointer-top-default"></div>
    </div>
    <div class="surround-md border-default rounded-md">
      <button type="button" class="menu-button-sm button-primary align-left">Item 1</button>
      <button type="button" class="menu-button-sm button-primary align-left">Item 2</button>
      <button type="button" class="menu-button-sm button-primary align-left">Item 3</button>
      <button type="button" class="menu-button-sm button-primary align-left">Item 4</button>
      <button type="button" class="menu-button-sm button-primary align-left">Item 5</button>
    </div>
  </div>
</div>

The first thing is to change the class menu-activator to menu-activator-dynamic - this makes the menu activate on hover instead of a click. On touch screen devices it will be converted to a tap. The next change is the addition of the pointer (lines 04 - 06). Here we just need a div with two classes, pointer-top-sm and pointer-top-default. The first class establishes that we are defining a pointer at a given size (in this case small), and the second makes a top pointer in the default theme colour. You could easily use a different theme colour for the pointer using another class such as pointer-top-accent-1, for example. Then we can wrap the pointer div just to give some left spacing and push it to the right a bit. Finally, the menu dropdown buttons have been wrapped with a surround with a border and rounding. The surround classes come in 3 flavours, surround-sm, surround-md, and surround-lg having 1px, 2px, and 3px padding respectively. Of course, there's no reason why you can't wrap the dropdown buttons with a container or whatever customisation you like.

Configuration options

There are two configuartion options available for menus which control animation transitions when dropdowns are shown and hidden. If you want to disable the fade-in and fade-out animations just set both dropdownFadeInDuration and dropdownFadeOutDuration to 0. The following code shows how to configure these options.

$( document ).ready(function() {
    
  ui.initMenus({
    dropdownFadeInDuration: 700,
    dropdownFadeOutDuration: 700
  });
        
});
