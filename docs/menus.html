About Webui

WebUI is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. WebUI has been build from the ground up with flexibility as the primary goal - even the components are made that way. WebUI is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Menus[Requires: webui.js]

WebUI places a lot of emphasis on the ability to create menus in the style that you want. As you will see, there are a lot of customisation options available. Menus do require the webui.js javascript file to be included in your project as well as the CSS, but you won't have to write any javascript except for setting configuration options as explained a bit later. First, let's take a look at some simple menus and then progress to the more advanced features.

<div class="content-row menu-bar-sm" role="navigation">
  <div class="content-col-20 accent-1">
    <div class="content-col width-lg text-xl content-col-align-lg col-spacing-left-md">
      <i class="fa fa-photo"></i>
    </div>
    <div class="content-col width-lg">
      <button id="Menu1" class="menu-activator menu-button-sm menu-button-accent-1 text-left">Option 1<i class="fa fa-caret-down col-spacing-left-xs"></i></button>
      <div class="dropdown-content menu-close width-xl accent-1">
        <button class="menu-button-sm menu-button-accent-1 background-hover-accent-2 text-left">Item 1</button>
        <button class="menu-button-sm menu-button-accent-1 background-hover-accent-2 text-left">Item 2</button>

        <button class="menu-button-sm menu-button-accent-1 background-hover-accent-2 text-left">Item 4</button>
        <button class="menu-button-sm menu-button-accent-1 background-hover-accent-2 text-left">Item 5</button>                     
      </div>
    </div>
    <div class="content-col width-lg">
      <button id="Menu2" class="menu-activator menu-button-sm menu-button-accent-1 text-left">Option 2<i class="fa fa-caret-down col-spacing-left-xs"></i></button>
      <div class="dropdown-content menu-close width-xl accent-1">
        <button class="menu-button-sm menu-button-accent-1 background-hover-accent-2 text-left">Item 1</button>

The code example above shows a basic menu for a typical desktop layout, where a number of fixed widths are used and we haven't added any media flags for responsive behaviour. WebUI allows the use of almost any html elements to build a menu, and you are not limited to buttons or lists. It is worth noting that WebUI includes menu button types that are specifically styled and sized to suit menu layouts.

Let's examine the important parts as follows:

menu-bar-sm - There are two menu-bar classes - menu-bar and menu-bar-sm. You don't always need them, but it's as well to add one of them, since they set a height and maximum height on the menu bar to match the height of corresponding controls that may be used. Of course, if you have your own custom elements that don't fit those dimensions, then you probably don't want it.

menu-activator - This class registers a control as the element that will be the trigger to open and close a menu. You can use just about any element as the menu activator.

dropdown-content - The dropdown-content class registers an element to be the container for any dropdown content. The dropdown content itself can be anything you want.

menu-close - Specifying menu-close on the dropdown container will make the dropdown close after a menu item such as a button or link is clicked or tapped. If you happened to have content other than buttons and links and you want to toggle it via script, you can do that by calling the toggleMenuItem WebUI function and passing in the menu activator element id or as an object.

So, what if we want a menu option near to the right of the menu and we want the dropdown to point from right to left? In this case we can just add the dropdown-right class to the dropdown content container.

Creating responsive menus

There are many styles of responsive menu, and one of the most popular for smaller screen sizes is the full width collapsible type. The menu simply expands and shrinks as you open and close menus options. Very often the menu will also push down any page content that may be displayed below it.

The following code will produce that kind of menu

<div class="content-row menu-bar" role="navigation">
  <div class="content-col-20 accent-1">
    <button id="menuButtonHamburgerMobile" class="menu-activator menu-button menu-button-accent-1 width-md-lg"><i class="fa fa-bars"></i></button>
    <div class="dropdown-content-expand width-full">      
      <button class="menu-activator menu-button-sm menu-button-dark align-left">Menu 1</button>       
      <div class="dropdown-content-expand menu-close accent-4 width-full">                                     
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 1</span></button>
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 2</span></button>
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 3</span></button>
        <button  class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 4</span></button>
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 5</span></button>
      </div>              
      <button class="menu-activator menu-button-sm menu-button-dark align-left">Menu 2</button>       
      <div class="dropdown-content-expand menu-close accent-4 width-full">
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 1</span></button>
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 2</span></button>
        <button class="menu-button menu-button-sm menu-button-accent-5 align-left"><span class="col-spacing-left-lg">Item 3</span></button>                                 
      </div>
    </div>
  </div>
</div>

On line 5 a the class dropdown-content-expand is added - this is actually the class that will make the menu push any page content down as you expand the menu. If you prefer the menu to overlay any page content you can replace just that single top most dropdown-content-expand with dropdown-content. There is a third kind of dropdown container called dropdown-sheet - this is used mainly for desktop sized screens and is often used on ecommerce sites.

NOTE: If you do use dropdown-content instead of dropdown-content-expand on the top most activator button for a mobile menu, you will need to add the menu-inclusive class as well, otherwise the menu will close when selecting sub menu activators. That's because dropdown-content on its own will close all other menus by default and would normally be used for the desktop style menu.

Customisation

OK, now let's do some customisation of our first menu. Say we want to have a white dropdown with a small arrow on top, and maybe a black border and make the corners a bit rounded. Oh, and let's make it dropdown on hover instead if the default click. No problem, taking the second menu section (lines 7 - 14) from our first example we can change it as follows:

<div class="content-col col-spacing-left-sm">
  <button id="Menu1" class="menu-activator-dynamic menu-button-sm menu-button-accent-5 text-left">Option 1<i class="fa fa-caret-down col-spacing-left-xs"></i></button>
  <div class="dropdown-content menu-close width-xl transparent">
    <div class="col-spacing-left-md">
      <div class="pointer-top-sm pointer-top-accent-5"></div>
    </div>
    <div class="border-sm border-accent-5 rounded-md">
      <button class="menu-button-sm menu-button-light background-hover-secondary text-left rounded-top-md">Item 1</button>
      <button class="menu-button-sm menu-button-light background-hover-secondary text-left">Item 2</button>
      <button class="menu-button-sm menu-button-light background-hover-secondary text-left">Item 3</button>
      <button class="menu-button-sm menu-button-light background-hover-secondary text-left">Item 4</button>
      <button class="menu-button-sm menu-button-light background-hover-secondary text-left rounded-bottom-md">Item 5</button>
    </div>
  </div>
</div>

The first thing is to change the class menu-activator to menu-activator-dynamic - this makes the menu activate on hover instead of a click. On touch screen devices it will be converted to a tap. The next change is the addition of the pointer (lines 4 - 6). Here we just need a div with two classes, pointer-top-sm and pointer-top-default. The first class establishes that we are defining a pointer at a given size (in this case small), and the second makes a top pointer in the default theme colour. You could easily use a different theme colour for the pointer using another class such as pointer-top-accent-1, for example. Then we can wrap the pointer div just to give some left spacing and push it to the right a bit. Finally, the menu dropdown buttons have been wrapped with a div tag that has a border and rounding.

Configuration options

There are two configuration options available for menus which control animation transitions when dropdowns are shown and hidden. If you want to disable the fade-in and fade-out animations just set both dropdownFadeInDuration and dropdownFadeOutDuration to 0. The following code shows how to configure these options. 
$( document ).ready(function() {
    
  ui.initMenus({
    dropdownFadeInDuration: 700,
    dropdownFadeOutDuration: 700
  });
        
});

Events

There are 4 event callbacks that can be used, ui.dropdown.show.before, ui.dropdown.show.after, ui.dropdown.hide.before, and ui.dropdown.hide.after. The following script shows how to setup the event callbacks.

$( document ).ready(function() {

  $( "#menu1" ).on("ui.dropdown.show.before", function(e) {
    // Take some action
  });

  $( "#menu1" ).on("ui.dropdown.show.after", function(e) {
    // Take some action
  });

  $( "#menu1" ).on("ui.dropdown.hide.before", function(e) {
    // Take some action
  });

  $( "#menu1" ).on("ui.dropdown.hide.after", function(e) {
    // Take some action
  });

});
