About Webui

Webui is a little different to most other CSS frameworks due to an emphasis on flexibility and creativity. Although most frameworks do attempt to provide flexibility, they often enforce a particular approach or style, and come with too many limitations. Webui has been build from the ground up with flexibility as the primary goal - even the components are made that way. Webui is a system that allows you to invent new designs, and provides simple building blocks to construct much larger CSS components. Complex data entry forms can be created in a precise layout using the built-in grid system that doesn't throw any surprises. Add responsive behaviour exactly where you want it, with the flexibility to create fixed non-responsive layouts or layout sections.

Be creative. Start something amazing today!

Setting up a modal

Webui allows you to easily create responsive model dialogs in 3 different sizes: small, medium, and large. Modal dialogs can be positioned in one of 9 different screen locations, these are: top left, top center, top right, middle left, middle center (default), middle right, bottom left, bottom center, and bottom right.

These modals are responsive out of the box, so the width and height will adjust according to the available screen area - this is so that a modal can never go off screen. As far as the modal content goes there is no special layout, so you can design your modal how you want it. However, it's always best to design your layout to suit the initial modal sizes. But if you need more height and you don't mind a vertical scrollbar, then you can add as much as you wish.

Here is an example modal with a heading, content area, and some buttons.

<div id="modal1" class="modal">
  <div class="modal-md modal-bottom-right modal-animate background-primary raised-sm">
    <div class="form-row text-lg">
      <div class="form-col-20 background-secondary">
        <div class="form-col-17 col-spacing-left-sm">
          Popups
        </div>
        <div class="form-col-3 col-spacing-right-sm text-right">
          <a href="javascript:void(0)"><i onclick="ui.hideModal($('#modal1'))" class="fa fa-close color-accent-2 color-hover-accent-3"></i></a>
        </div>
      </div>
    </div>
    <div class="form-row">
      <div class="form-col-20">
        <div class="area">
          <div class="row-spacing-xs"></div>
            <div class="content-row">
            <div class="content-col-20">
               Create responsive modals, and optionally add animation.
            </div>
          </div>
        </div>
      </div>
    </div>
    <div class="col-spacing-right-md place-bottom-right">
      <button type="button" class="button-sm button-info width-md">Save</button>
      <button type="button" onclick="ui.hideModal($('#modal1'))" class="button-sm button-accent-4 width-md">Cancel</button>
      <div class="row-spacing-sm"></div>
    </div>
  </div>
</div>
Let's go through the code above. The first line is where the wrapper is defined for the modal - that's actually the overlay that you see covering the page behind. Then on line 2, the actual modal dialog is defined at the size we want, in this case the class modal-md is a medium size. There's also a positional class on the same line, modal-bottom-right, which means the modal will open near to the bottom right of the browser window. You may notice the modal-animate class also, which just makes the modal fade in and out. There is also a background color set using the primary theme color, in this case it's white. The rest of the markup is entirely your choice, but it's worth taking a closer look at how we open and close the modal and at some of the classes used in the above example.

So to open the modal it's just a case of adding an id to the outer wrapper and calling ui.showModal and passing in the id or the object. You will also see that ui.hideModal is used for closing the modal.

NOTE:

It is important that the functions ui.showModal and ui.hideModal are used to open and close modal dialogs as some background tasks need to be completed, such as disabling and re-enabling scrolling for the underlying page.

In the above example, there are a few spacing and positioning classes of interest. On line 6 the class col-spacing-left-sm is used - this just moves the text away from the left of the heading, and we do something similar for the small X button in the top right. Finally, an important class used here is place-bottom-right - this positions the buttons at the bottom right of the modal, but some space is needed between the buttons and the bottom edge, so just add some row spacing. If the content of the modal is longer than the modal itself, the positioned buttons will stay exactly where you put them relative to the modal view window. But a better way is to make a footer area to house the buttons, and you can easily add a scrolling body section in the middle by using the class scroll-y

$( document ).ready(function() {
    
    $("#commandShowModal1").click( function() { 
        ui.showModal("#modal1");
    });
        
});

This is how the markup above would look in the browser.

Configuration

There is one configuration option to enable or disable the ability to close the modal by clicking or tapping outside the modal on the backdrop. By default it is turned off, but the following shows how to enable it. 

$( document ).ready(function() {
    
  $("#modal1").modalControl({
    closeFromBackdrop: true
  });
        
});

Events

There are 4 event callbacks that can be used, ui.modal.show.before, ui.modal.show.after, ui.modal.hide.before, and ui.modal.hide.after. The following script shows how to setup the event callbacks.

$( document ).ready(function() {

  $( "#modal1" ).on("ui.modal.show.before", function(e) {
    // Take some action
  });

  $( "#modal1" ).on("ui.modal.show.after", function(e) {
    // Take some action
  });

  $( "#modal1" ).on("ui.modal.hide.before", function(e) {
    // Take some action
  });

  $( "#modal1" ).on("ui.modal.hide.after", function(e) {
    // Take some action
  });

});
